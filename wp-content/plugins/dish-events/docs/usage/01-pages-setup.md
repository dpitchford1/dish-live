# Pages Setup — WordPress Pages Required by the Plugin

The plugin needs a set of WordPress pages to host its shortcodes. Before the site works end-to-end, these pages need to be created and linked to the plugin in **Settings → Pages**.

---

## Step 1 — Create the Pages

Create each page listed below in **Pages → Add New**. The page title and slug are suggestions — use whatever fits your site's navigation.

| Page | Suggested title | Suggested slug | Shortcode to add |
|---|---|---|---|
| Classes listing | Classes | `/classes` | `[dish_classes]` |
| Chefs listing | Our Chefs | `/chefs` | `[dish_chefs]` |
| Checkout | Book a Class | `/book` | `[dish_booking]` *(coming soon)* |
| Booking details | Booking Confirmed | `/booking-details` | `[dish_booking_details]` *(coming soon)* |
| Enquiry / Contact | Contact Us | `/contact` | *(no shortcode — your existing contact page)* |
| My Account | My Account | `/my-account` | `[dish_profile]` |
| Login | Sign In | `/sign-in` | `[dish_login]` |
| Register | Create Account | `/create-account` | `[dish_register]` |

> **Checkout and Booking Details** — these shortcodes are not yet active. Create and save the pages now so they're ready; they'll automatically work once the shortcodes are activated in a future release.

> **Enquiry page** — this is your existing contact page. No new page needed; just point the plugin at whatever page you use for enquiry forms.

---

## Step 2 — Add the Shortcodes

For each page, place its shortcode as the **only content** in the page body. These shortcodes generate all their own markup — you don't need to add headings, intro text, or anything else unless your theme template already handles that.

**Example — the Classes listing page:**
```
[dish_classes]
```

If you'd like to show the calendar by default instead of the grid:
```
[dish_classes view="calendar"]
```

---

## Step 3 — Assign Pages in Settings

Once the pages are published, go to **Dish Events → Settings → Pages** and assign each page to its role using the dropdown.

| Setting label | Page to select |
|---|---|
| Classes listing page | Your Classes page |
| Checkout page | Your Book a Class page |
| Booking details page | Your Booking Confirmed page |
| Enquiry / Contact page | Your Contact Us / enquiry page |
| My account page | Your My Account page |
| Login page | Your Sign In page |
| Register page | Your Create Account page |
| Chefs listing page | Your Our Chefs page |

Click **Save Changes** after assigning all pages.

> These assignments tell the plugin where to direct "Book Now" buttons, "My Account" links, and post-booking redirects. If a page is left unassigned, those links will have no destination and may break the booking flow.

---

## Formats Archive — No Setup Required

Class formats get their archive pages automatically. Each format (e.g. "Hands On") gets a public URL at `/classes/hands-on/` that lists all published templates in that format.

No shortcode or page setup is needed for these — they're generated by the plugin's taxonomy archive routing.

---

## Upcoming Menus — Optional Page

If you want a public menus listing page, create an additional page and add:

```
[dish_upcoming_menus]
```

Or with a custom limit:

```
[dish_upcoming_menus limit="20"]
```

This page doesn't need to be assigned in Settings — it's standalone.

---

## After Setting Up Pages

Once pages are created and assigned:

1. Go to **Settings → Permalinks** and click **Save Changes** to flush WordPress rewrite rules. This ensures all the plugin's URLs (class template pages, chef pages, format archives) resolve correctly.
2. Visit each page on the frontend to verify the shortcode output renders.
3. Confirm the **Book Now** button on a class page redirects correctly to your checkout page.
